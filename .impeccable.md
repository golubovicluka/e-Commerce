# Impeccable — Design Context for Shoply

## Design Context

### Users
Consumer shoppers in Serbia browsing and buying tech (laptops, phones, tablets,
peripherals) on desktop and mobile. They compare prices, care about installment
options (12/24/36 months), and want a fast path from discovery to checkout.
Prices are in RSD everywhere — never `$`.

### Brand Personality
Warm, confident, editorial. Three words: **crafted, honest, unhurried**.
The shop should feel like a well-set magazine spread about technology, not a
fluorescent-lit warehouse. Emotional goals: trust and quiet desire — "I want
this and I believe these people."

### Aesthetic Direction
- **Theme**: light. Warm paper background (`#f0eee8` family) — daytime browsing
  product. One inverted "ink" surface (warm near-black) reserved for the two
  highest-impact moments: the hero product stage and the newsletter band.
- **Accent**: a single clay/vermilion (`#d9622b`), used sparingly (CTAs, focus,
  active states). 60/30/10 weighting: paper surfaces / text+borders / clay.
- **Type**: Syne (display, 700–800, tight tracking) + DM Sans (body) +
  JetBrains Mono (prices, SKUs, EAN, counts). These are the established brand
  fonts of this codebase — keep them; do not re-skin.
- **Anti-references**: generic admin dashboards, purple-gradient AI slop,
  centered-everything card grids, glassmorphism for its own sake.

### Design Principles
1. **Editorial, not templated** — left-aligned eyebrow + display heading
   section pattern; asymmetric compositions; oversized type is the decoration.
2. **Prices are typography** — every price set in JetBrains Mono; the number
   is the hero of any commerce surface.
3. **Ink is rare** — inverted panels appear only as bookends (hero stage at
   the top, newsletter band at the bottom) so they stay powerful.
4. **Motion = entrance + feedback only** — staggered reveals on page load,
   physical hover lifts, heart-pop on wishlist; transform/opacity only;
   `prefers-reduced-motion` always respected.
5. **Never color alone** — stock/status always pairs a dot or icon with text.

### Constraints
- Angular 15 + PrimeNG 15 + PrimeFlex 3 (utility classes are PrimeFlex, NOT
  Tailwind — `md:col-6`, not `md:grid-cols-2`).
- Design tokens live in `src/styles/tokens.scss`; consume via `var(--token)`,
  never hardcode hex/fonts in component styles.
- Unit tests assert component class APIs, not templates — keep inputs/outputs
  and public methods stable when restyling.
